Lesson 1: The Basics of the Web and HTML

How the Web Works

The five main agents in the process of using the web are:

  1. The user
  2. The web browser
  3. The internet
  4. HTTP protocol
  5. Servers

The user opens a web browser. The user commands the browser through an url or a search query. this data form a package which travel over the internet ("the world's largest computer network"). HTTP, an applications protocol, takes the signals and sends them to servers that host the content. The servers then send information back along the chain of command (through HTTP through the internet through my browser) back to the user. To explain this concepts I have use in my classes this video: Como Funciona el Internet - Documental Audio Latino .

HTML

HTML mean Hypertext Markup Language. HTML documents contain the data that the user view in a web page and the format of how the data is presented. This type of document contains:

  • Text content (what you see)
  • Markup (what it looks like)
  • References to other documents (e.g. images and videos)
  • Links to other pages
  • Meta Data
Tags vs. Elements vs. Attributes

An element in HTML represents some kind of structure or semantics and generally consists of a start tag, content, and an end tag. A tag is used to mark up the start and end of an HTML element. A start tag consists of an opening angle bracket followed by the element name, zero or more space separated attribute/value pairs, and a closing angle bracket. A good reference about the tags supported is w3schools.com tags .

An attribute defines a property for an element, consists of an attribute/value pair, and appears within the element's start tag. An element's start tag may contain any number of space separated attribute/value pairs. Also each tag support some specific attributes, HTML Attribute Reference shows the defined attributes for html and if you give clik you will be find which tags support it.

Why Computers are Stupid

Computers are sensitive to typos, and small mistakes such as missing an End Tag will affect the page layout! Developers must then "debug" to track and correct their mistakes.

Testing and Quality of the code is an important consume a big part of developing programs or web page.

Inline vs Block Elements

HTML elements are either inline or block.

Block elements always make "enter", so create a box for the inside content. Examples of Block Elements: p, div, ul, li, and h1.

Inline elements are inline with the other elements. Examples of Inline Elements: a, span, b and em .

Lesson 2: Creating a Structured Document with HTML

Concepts

HTML, CSS, Javascript : 3 languages serving different purposes when developing a webpage.

Using the analogy of a house...

  • HTML: HyperText Markup Language = the standard markup language used to create web pages. HTML creates structure in a webpage (think layout of a house)
  • CSS: Cascading Style Sheets = language used for describing the look and formatting of a webpage. CSS creates style in a webpage, such as font size, colours, thickness of borders etc (think decoration within the house).
  • JavaScript = language used to create interactive components in a webpage (think of remotes or garage doors needed to perform actions)

In this section we are going to talk about HTML.

Document Object Model (DOM)

Document Object Model: a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

When programmers talk about the DOM, they are talking about the tree-like structure of a webpage. In a browser, this structure shows up as a series of nested boxes.

The relationship between indented HTML and boxes

Everything on a webpage is organized in "Boxes" - or rectangular sections. This gives developers the flexibility to rearrange content easily on a webpage, and helps them "Boxify, or break down a given graphic design into various boxes.

Each HTML "box" has 4 elements: margin, border, padding, content. With 4 components to each box, it can often be hard to get the accurate size of a box. Use two techniques to deal with sizes:

  1. Set sizes in terms of percentages rather than pixels.
  2. Set the box-sizing attribute to border-box for every element.

Lesson 3: Adding CSS Style to HTML Structure

What is CSS?

CSS stands for Cascading Style Sheets. CSS controls the style of the webpage. The "cascading" part means that CSS rules are applied not only to the elements they directly match (via classes), but also to all of those elements' child elements (i.e. inheritance). However, if a child element has multiple overlapping rules defined for it, the more specific rule takes effect.

Inheritance prevents certain properties from being declared over and over again in a style sheet. It is a fundamental building block for the abstract thinking portion of a programmer's thought process.

Tips

Avoiding repetition by leveraging CSS is important because:

  1. It saves the programmer time.
  2. Less code needs to be written, therefore making it easier to understand when it is being reviewed.
  3. Less risk of typos and therefore less chance of buggy code
  4. Each line that you wirte will travel from server to client, so save lines of code mean save bandwidth.
